# Push_swap

## Description

<!-- 2〜4文で十分: 何をするプロジェクトか -->
2つのスタックと限られた操作（sa, pb, ra など）だけを使い、整数列を最小の操作数でソートする操作列を出力するプログラム。
入力の乱れ具合（disorder）に応じて4つの戦略を切り替える。

## Instructions

```bash
make            # push_swap をビルド
./push_swap 2 1 3 6 5 8                  # 操作列を stdout に出力
./push_swap --simple 5 4 3 2 1           # O(n^2) 戦略を強制
./push_swap --medium 5 4 3 2 1           # O(n√n) 戦略を強制
./push_swap --complex 5 4 3 2 1          # O(n log n) 戦略を強制
./push_swap --adaptive 5 4 3 2 1         # disorderで自動選択（デフォルト）
./push_swap --bench <args> 2> bench.txt  # ベンチマークを stderr に出力
make clean / fclean / re
```

検証例:

```bash
ARG="4 67 3 87 23"; ./push_swap $ARG | ./checker_linux $ARG   # → OK
```

## Algorithms


### Complex — O(n log n)
**数を2進数として見て、下の桁から1ビットずつ「0のグループ」と「1のグループ」に
仕分けることを繰り返すソート**。比較（どっちが大きい？）を一切せず、
各要素のビットだけを見て機械的に動くのが特徴。

処理は2段階：

1. **前処理（ランク化）**: 各値を「小さい方から何番目か」(0〜n-1) に置き換える
2. **本体（ビットごとの仕分け）**: bit 0 から最上位ビットまで、1ビットにつき1パス
---
1. **前処理（ランク化）**

何をするか

入力の各値を「昇順で何番目か」に置き換える。

入力:   42  -3  17   8
ランク:  3   0   2   1     (-3が0番目、8が1番目、17が2番目、42が3番目)

以後は元の値のことは忘れて、ランクの列をソートする。
ランクが昇順に並べば元の値も昇順に並ぶ（順序関係が同じだから）。

なぜ必要か（2つの理由）

理由1: 負の数対策。
Cのintは2の補数表現なので、-3のビット列は 111...101。
ビットで仕分けると負の数が正の数より「大きい」扱いになり、結果が壊れる。
ランクは必ず 0 以上なのでこの問題が消える。

理由2: パス数の削減（こちらが本質）。
パス数 = 扱う数の最大ビット数。
生のintのままだと最大32パス必要だが、ランクは 0〜n-1 なので
n=500 でも最大値499 = 9ビットで済む。

操作数の概算 (n=500, 1パス ≈ 仕分けn回 + 戻し平均n/2回 ≈ 750操作):
  ランク化なし: 32パス × 750 ≈ 24,000操作 → 合格ライン12,000を超え不合格
  ランク化あり:  9パス × 750 ≈  6,750操作 → 余裕で合格

ランク計算は操作列を出力しない内部計算なので、操作数のコストはゼロ。
タダでパス数を1/3以下にできる。

実装


入力配列のコピーを作る（元の並び順を保存するためコピーが必要。
直接ソートすると「元のi番目の値のランク」を調べる相手が消える）
コピーを昇順ソート（内部計算なのでバブルソートで十分）
各値について「ソート済みコピーの何番目か」を線形探索し、その添字で上書き

本体：1ビットずつ仕分ける

1パスの動き（ビット i を見るパス）

aの要素数 n をパス開始前に控えておき、n回くり返す:
    aの先頭のランクの iビット目 を見る
        0 なら → pb（bへ送る）
        1 なら → ra（aの底へ回す）
n回終わったら:
    bが空になるまで pa（全部aへ戻す）

これを i = 0（最下位ビット）から最大ビット数まで繰り返すと、aが昇順になる。

iビット目の取り出しは (rank >> i) & 1：
i桁ぶん右シフトして、一番下の桁だけを & 1 で取り出す。




計算量（push_swap操作モデル）


パス数: ⌈log₂ n⌉ （ランク化のおかげ。n=500 → 9パス）
1パスあたり: 仕分けに n 回（pbかra）+ 戻しに最大 n 回のpa ≤ 2n
合計: O(n log n) 操作
空間: スタック2本 + ランク計算の一時配列 = O(n)


実測（乱数入力）:

n操作数ベンチマーク基準100約1,100< 1,500 (good)5006,784< 8,000 (good) ※課題PDFの使用例と同じ値

操作数は入力の並び順によらず一定という面白い性質がある。
仕分けの内訳は「ランク0〜n-1の各ビットに0と1が何個ずつあるか」だけで
決まり、それは並び順に関係なく固定だから。
